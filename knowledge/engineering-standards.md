# Engineering Standards

## Philosophy
Build reliable, maintainable software that creates business value.

## Standards
- Thin controllers
- SOLID where appropriate
- Readable code
- Tests for business-critical functionality
- Meaningful commits
- Secure by default
- Measure before optimizing

## Change Safety
Before completing any task:
- Confirm existing functionality still works.
- Review adjacent features.
- Consider API, DB and integration impacts.
- Update tests where appropriate.
- Never introduce regressions knowingly.

## AI Notes
- Respect the existing architecture.
- Do not reorganize projects during unrelated work.
- Recommend refactors separately.
- Understand before implementing.

## Universal Checklist
- Requirements met
- Existing behaviour preserved
- Tests updated
- Documentation updated
- Security reviewed
- Performance acceptable
- No unnecessary complexity
